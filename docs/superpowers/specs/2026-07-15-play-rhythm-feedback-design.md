# 游戏节奏与反馈闭环设计

## 问题与根因

当前玩法已经包含弹弓、五类糖果球、机关、连击和 25 个关卡，但一次发射无论真实状态如何都会等待固定 5200ms 才执行 `endShot()`。空射或球提前停住时，玩家只能等待；固定时长也无法正确表达分裂球等多球场景的结束条件。

碰撞链路会分别创建得分文字、连击文字、顶部反馈和糖果狂热横幅。同一次碰撞触发多个规则时，这些对象同时出现并互相遮挡。技能按钮只显示符号，玩家无法从按钮本身判断当前能力。`GameSave.soundEnabled` 已经存在，但没有任何声音实现。

纯视觉增强无法解决等待和回合误判；大规模重写物理与关卡风险过高。确认采用“核心节奏闭环”方案，在保留现有玩法和存档的前提下重做回合状态判断与反馈编排。

## 回合状态

新增独立纯规则模块 `shotLifecycle.ts`，统一维护最短飞行时间、低速阈值、连续稳定时间、出界范围和最大飞行兜底。`GameScene.update()` 每帧把所有有效糖果球的位置、线速度和角速度交给规则模块：

- 任一糖果球仍在有效区域高速运动时，稳定计时清零。
- 所有糖果球持续低速 450ms，且已经越过最短飞行保护时间后，进入回合收尾。
- 所有糖果球都已离开扩展后的画面边界时，不等待低速计时，直接进入回合收尾。
- 分裂球必须等待三个球全部低速或全部出界；不能只看主球。
- 达到最大飞行时间时兜底结束，避免异常刚体永久阻塞。

`endShot()` 保持幂等，只负责清理当前球和技能状态。若仍有目标且还有糖果球，使用约 300ms 的短过渡再生成下一球；期间继续允许已释放结构自然运动和目标掉落结算。目标已清空或球数耗尽时继续走现有结果页，延迟回调在场景退出后不得生成残留球。

按上述时间预算，球停止 450ms 后确认稳定，再经过约 300ms 生成下一球，总等待约 750ms，低于 1.2 秒验收线。

## 反馈编排

顶部反馈收敛为单一消息通道。消息带普通、重要、庆祝三个优先级和短暂保护时间，低优先级消息不能立即覆盖“宝箱打开”“糖果狂热”等关键结果。每次显示使用一次轻微缩放与淡入，保证命中当帧即可看到变化。

得分和连击不再分别创建多个文字对象。`showScorePop()` 维护一个当前聚合提示：短时间内连续发生的得分累加到同一数字，并在连击达到 2 次后追加 `连击 xN`。粒子、积木形变和镜头震动仍保留，因此聚合文字不会削弱命中位置的可见反馈。

糖果狂热继续保留独立庆祝，但横幅尺寸缩小并避开顶部 HUD；奖励分数合并到当前得分提示，不再额外创建第二个重叠数字。

## 技能交互

技能按钮保留右下角固定位置和稳定尺寸，图标下方增加能力短名称：冲刺、坠击、跃起、引爆、分裂。按钮外围增加随飞行时间递减的进度环，用视觉方式表达当前球仍可使用技能；不增加教学段落或弹窗。

技能触发后立即关闭交互和循环脉冲，播放专属反馈，并保留现有物理效果。回合进入收尾后同样隐藏按钮，避免玩家点击即将销毁的球。

## 本地音效

新增 `CandyAudio`，使用浏览器 Web Audio API 程序化合成短音效，不下载文件、不访问网络。音频上下文只在玩家首次手势触发时创建或恢复，以兼容 iPad 的自动播放限制。

声音分为发射、普通命中、连击、技能、爆炸、胜利和失败七类短提示。高频碰撞使用最小播放间隔限制，避免一次连锁碰撞产生刺耳叠音。`BootScene` 根据存档创建共享音效实例；菜单和游戏 HUD 都提供声音图标开关，切换后立即更新 registry、localStorage 和音效实例。

音频初始化失败时只降级为静音，不影响发射、结算或存档。

## 关卡与兼容范围

保留现有 25 关的编号、名称、章节、目标和特殊机制。关卡调整只针对实玩确认的阻塞点：优先减少早期关卡中遮挡目标的高耐久材料，或增加少量容错球数；不通过降低全部目标耐久度破坏宝箱和分裂球的教学节奏。

存档继续使用 `tantanle-save-v1` 和 `version: 1`。新增运行时状态不写入存档，旧存档可以直接加载。

## 异常与边界

- 场景结束、重开或返回选关后，所有回合延迟、反馈 tween 和活动球都必须停止或失效。
- 低帧率下按 `delta` 累计稳定时间，但单帧增量需要限制，防止页面从后台恢复时立即判定回合结束。
- 非有限速度视为仍未稳定，直到最大时长兜底。
- 糖果球接触地面后缓慢滚动不会永久占用回合；高速反弹和传送后的球不会被提前回收。
- 过关庆祝已经启动时，不再生成下一球；最后一球失败时只进入一次结果页。
- 音频上下文不可用、被系统暂停或静音时，游戏保持完全可玩。

## 测试策略

1. 先补 `shotLifecycle` 纯规则失败测试，覆盖低速累计、运动重置、多球、出界、非有限速度、后台大 `delta` 和最大时长兜底。
2. 补源码契约测试，确认固定 5200ms 延迟已移除、`update()` 驱动动态回合状态、下一球延迟受场景状态保护。
3. 补声音偏好测试，确认开关只修改 `soundEnabled` 且保留所有关卡进度。
4. 运行全量 Vitest 与 TypeScript/Vite 构建。
5. 在浏览器连续实玩至少 3 关，覆盖命中、空射、技能、失败重开和过关下一关。
6. 分别在桌面和 iPad 834x1194 视口截图检查 HUD、技能按钮、命中提示、糖果狂热和结果页；记录控制台错误并区分开发环境 HMR 噪声与游戏错误。

## 非目标

本轮不新增账号、广告、内购、排行榜、统计埋点或后端；不改变 GitHub Pages 部署；不新增第 26 关以后内容；不整体拆分 `GameScene`。大规模场景美术替换和全新玩法章节应在节奏稳定后单独设计。
