# 关卡目标与三星冲刺设计

## 问题与目标

当前主循环已经具备清晰的发射、命中、技能和回合收尾，但“为什么继续打”仍主要依赖玩家自行理解。游戏 HUD 只显示分数、剩余球和糖果连锁，关卡的 `starScores` 直到结果页才体现；目标数量也只存在于场景数据中。儿童玩家因此难以判断当前完成度，也感受不到分数跨档的阶段性胜利。

本轮目标是在不增加操作负担的前提下补齐短目标闭环：随时知道还剩几个目标、距离下一颗星有多远、这一局有什么值得记住的表现。

## 方案选择

确认采用“关卡目标 + 三星冲刺 + 胜利勋章”方案。单纯增加爆炸和慢镜头只能放大已有反馈；新增球种和机关会增加学习成本并扩大关卡调整面。目标闭环可以复用现有关卡数据与计分规则，覆盖所有 25 关，并直接改善每一局的可理解性和继续动力。

## 纯规则模块

新增 `levelProgress.ts`，集中承载不依赖 Phaser 的展示规则：

- 根据 `score` 和三个星级阈值计算当前星数、最高目标分数占比、下一颗星所需分数。
- 比较前后星数，返回本次新获得的星数。一次跨越多档时只返回一个最高档位文案。
- 根据胜负、真实发射次数、目标完成数、目标总数和是否新纪录，生成结果页表现文案。

纯规则不修改分数，也不写存档。`calculateStars()` 仍是星级最终口径，`levelProgress.ts` 复用它或保持完全一致的边界判断。

## 游戏 HUD

`GameScene` 增加两个运行时状态：

- `shotsUsed`：每次有效发射后加一，糖果狂热奖励球不会修改它。
- `earnedStars`：当前已经在 HUD 中点亮的星数，用于识别新的星级跨档。

现有左上 HUD 的第二行加入目标完成度，例如“目标 1/3”。左侧空余区域增加三星进度条，右侧现有糖果连锁条保持位置不变。进度条包括白色底轨、深色轮廓、黄色斜纹填充和三颗带深色描边的星形标记；即使无法区分部分颜色，也能通过位置、明暗、纹理和星形状态识别。

每次 `refreshHud()` 根据当前分数重绘进度。若星数增加，所有新获得的星同时点亮，并只显示一条最高档位反馈：“第一颗星到手”“两星达成”或“三星达成”。动画使用约 140ms 的放大回弹和本地 `combo` 音效，满足 150ms 内反馈要求。目标清除、积木得分、机关得分和糖果狂热奖励都走同一刷新入口。

## 结果页

`finish()` 在更新存档前读取旧 `bestScore`，计算本局是否为新纪录，并把以下运行时数据传给 `ResultScene`：

- `shotsUsed`
- `targetsCleared`
- `totalTargets`
- `isNewBest`
- 更新后的 `bestScore`

胜利时，一发完成显示“一发制胜”，2-3 发显示“N 发漂亮过关”，更多发显示“N 发完成目标”；新纪录作为第二条短标签显示。失败时标题改为“差一点”，表现文案显示“还差 N 个目标”，让孩子知道下一次需要完成什么。结果页按钮和下一关流程保持不变。

本轮不增加剩余球奖励或额外勋章分数，避免改变既有星级阈值与历史最佳分数口径。

## 数据流与兼容

关卡进入时从 `LevelDefinition.targets.length` 得到目标总数；每次 `clearTarget()` 更新 `targetsCleared` 后刷新 HUD。所有得分入口继续调用现有 `refreshHud()`，因此星级进度不会漏掉炸弹、弹力垫、传送门或连击奖励。

`shotsUsed` 和 `earnedStars` 只存在于当前场景。存档仍使用 `version: 1` 和 `tantanle-save-v1`，旧进度可以直接加载。GitHub Pages、PWA Service Worker 和隐私口径不变。

## 异常与边界

- 分数一次跨越多颗星时只播放一次音效、显示一条文案，但所有对应星标都点亮。
- 分数达到阈值时按“包含边界”计算，与 `calculateStars()` 一致。
- 失败页不显示星级里程碑或新纪录；历史最佳只在胜利后更新。
- 糖果狂热增加 `shotsLeft` 时不减少或重算 `shotsUsed`。
- 最后一关没有下一关按钮，现有结果页分支保持不变。
- 场景重开后所有运行时计数归零，不复用上一局的星级动画状态。

## 测试策略

1. 先补纯规则失败测试，覆盖阈值边界、跨多星、下一星差值、一发制胜、普通胜利、新纪录和失败剩余目标。
2. 补场景契约测试，确认有效发射递增 `shotsUsed`、糖果狂热不修改它、目标清除刷新 HUD、结果数据完整传递。
3. 运行全量 Vitest 和 TypeScript/Vite 构建。
4. 浏览器连续实玩前 3 关，验证目标计数、跨星反馈、结果文案和下一关流程。
5. 在桌面和 iPad 834x1194 视口截图检查左侧三星进度与右侧糖果连锁不重叠。

## 非目标

本轮不新增玩法对象、关卡或资源文件，不修改物理、计分、难度、存档格式和部署流程，也不整体拆分 `GameScene`。场景文件拆分将在玩法状态稳定后单独设计。
